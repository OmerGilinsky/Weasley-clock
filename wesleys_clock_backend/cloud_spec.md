# Weasley Clock - Cloud & Database Specification

This document serves as the **Single Source of Truth (SSOT)** for the Cloud Functions, HTTP APIs, and Firestore Database schemas of the Weasley Clock IoT project. It ensures full alignment between the Flutter mobile application, the ESP32 physical clock firmware, and the Firebase serverless backend.

---

## 🗄️ Firestore Database Schema

### 1. `users` Collection
*Each document in this collection represents a family member registered to the clock.*
* **Document ID**: `uid` (Firebase Authentication User ID)
* **Fields**:
  | Field Name | Type | Description |
  | :--- | :--- | :--- |
  | `uid` | `string` | Unique user ID. |
  | `name` | `string` | Display name of the user (e.g., "Dad", "Mom"). |
  | `handNumber` | `number` \| `null` | Assigned clock hand (1, 2, 3, or 4). `null` if unassigned. |
  | `currentLocation` | `string` | Location name (e.g., "Home", "Work", "Unknown", "Stale"). |
  | `targetAngle` | `number` | Calculated motor angle (0–360) corresponding to `currentLocation`. |
  | `displayGreetingUrl` | `string` \| `null` | URL to a doodle/avatar image to display on LCD. |
  | `lastUpdated` | `timestamp` | Timestamp of last location update or GPS ping. |
  | `lastPromptedTime` | `timestamp` \| `null` | Timestamp of the last direct "Where are you?" prompt received. |
  | `enableReminders` | `boolean` | User preference to receive automatic daily reminders. |

---

### 2. `locations` Collection
*Defines the physical landmarks mapped to specific physical angles on the clock dial.*
* **Document ID**: Auto-generated by Firestore (e.g., `loc_abc123`)
* **Fields**:
  | Field Name | Type | Description |
  | :--- | :--- | :--- |
  | `id` | `string` | Unique location ID. |
  | `locationName` | `string` | Display name (e.g., "Technion", "Home", "Work"). |
  | `angle` | `number` | The exact angle (0–359 degrees) on the physical clock face. |

---

### 3. `voice_messages` Collection
*Audio bytes sent between family members, queued dynamically based on presence.*
* **Document ID**: Auto-generated by Firestore
* **Fields**:
  | Field Name | Type | Description |
  | :--- | :--- | :--- |
  | `id` | `string` | Unique message ID. |
  | `senderId` | `string` | `uid` of the sender. |
  | `recipientId` | `string` | Target user's `uid`, or `"all"` for a global family announcement. |
  | `audioUrl` | `string` | Firebase Storage HTTPS download URL for the audio file. |
  | `status` | `string` | `"queued"` (recipient away), `"ready_to_play"` (ready for ESP32), `"played"`. |
  | `createdAt` | `timestamp` | Time the voice message was created. |

---

### 4. `system_status` Collection
*Monitors the operational state of the physical ESP32 clock hardware.*
* **Document ID**: `main_clock` (or unique clock hardware ID)
* **Fields**:
  | Field Name | Type | Description |
  | :--- | :--- | :--- |
  | `status` | `string` | Current status: `"healthy"`, `"error"`, `"offline"`. |
  | `lastHeartbeat` | `timestamp` | Timestamp of the last successful ping from the ESP32. |
  | `speakerStatus` | `string` | Audio hardware status: `"connected"`, `"disconnected"`, `"error"`. |
  | `motorStatus` | `string` | Motor status: `"healthy"`, `"jammed"`, `"calibrating"`. |
  | `lastErrorCode` | `string` \| `null` | Exact code for debugging hardware failures (e.g., `MOTOR_2_STUCK`). |

## ⚡ Cloud Functions Specification (Firebase v2)

### 📄 Part 1: Event-Driven Triggers (Firestore Triggers)

#### 1. `onUserCreated` (Document Create Trigger)
* **Path**: `/users/{uid}`
* **Trigger**: A new user registers via Flutter.
* **Logic**:
  1. Runs inside a **Firestore Transaction** to prevent race conditions.
  2. Queries all current documents in `/users` to see which `handNumber` values (1, 2, 3, 4) are currently assigned.
  3. Allocates the first available number (e.g., if 1 and 2 are taken, assigns 3).
  4. If all 4 hands are occupied, assigns `null` and logs a warning.
  5. Updates `/users/{uid}` with the selected `handNumber`.

#### 2. `onUserLocationChanged` (Document Update Trigger)
* **Path**: `/users/{uid}`
* **Trigger**: `currentLocation` or GPS coordinates change.
* **Logic**:
  1. **Optimization Guard**: Immediately exits if the *new* `currentLocation` is equal to the *old* `currentLocation` (saving read costs).
  2. If the new location is `"Unknown"`, sets `targetAngle` to `0` (or default "Unknown" angle) and exits.
  3. Queries the `locations` collection to find the document matching the new `currentLocation` name.
  4. If found, writes the exact `angle` to `targetAngle` on the user document.
  5. If not found (unrecognized place), sets `targetAngle` to `0` (default) and updates `currentLocation` to `"Unknown"`.
  6. **Dynamic Greeting Playback Trigger**: Checks if the new location is `"Home"`. If yes, triggers `triggerQueuedMessagesOnArrival(uid)`.

#### 3. `onUserDeleted` (Document Delete Trigger)
* **Path**: `/users/{uid}`
* **Trigger**: Admin deletes a user.
* **Logic**:
  1. Deletes all associated files in Cloud Storage under the folder `audio_bites/{uid}/*`.
  2. Deletes all document records in `/voice_messages` where `senderId == uid` or `recipientId == uid`.
  3. Frees the `handNumber` associated with the user, making it instantly available for the next registration.

#### 4. `onLocationDeleted` (Document Delete Trigger)
* **Path**: `/locations/{locId}`
* **Trigger**: Admin removes a landmark.
* **Logic**:
  1. Identifies the deleted `locationName`.
  2. Queries all users whose `currentLocation` matches the deleted name.
  3. Uses a `writeBatch()` to atomicly update all matched users:
     * `currentLocation` ➔ `"Unknown"`
     * `targetAngle` ➔ `0`

#### 5. `onLocationUpdated` (Document Update Trigger)
* **Path**: `/locations/{locId}`
* **Trigger**: Admin changes a location's angle or name.
* **Logic**:
  1. **Optimization Guard**: Exits if `angle` and `locationName` are unchanged.
  2. If the `angle` changed, queries all users whose `currentLocation` matches the modified landmark.
  3. Updates their `targetAngle` in real-time, causing the clock motors to synchronize immediately.

#### 6. `onVoiceMessageCreated` (Document Create Trigger)
* **Path**: `/voice_messages/{msgId}`
* **Trigger**: A user records a voice message.
* **Logic**:
  1. Reads the `recipientId` from the message payload.
  2. If `recipientId == "all"`, checks if **any** user is currently at `"Home"`.
  3. If `recipientId` is a specific user, checks if *that specific user* is currently at `"Home"`.
  4. If the conditions are met (recipient present):
     * Updates status to `"ready_to_play"`.
  5. If the target recipient is away:
     * Updates status to `"queued"`.

#### 7. `onVisualGreetingCreated` (Document Create Trigger)
* **Path**: `/visual_greetings/{greetingId}`
* **Trigger**: User uploads a doodle / sketch.
* **Logic**:
  1. Links the image to the user.
  2. Updates `displayGreetingUrl` in the user's document.
  3. The ESP32 (listening to users) immediately displays the drawing on the physical LCD screen.

### 📄 Part 2: Time-Based / Scheduled Functions (Cron Triggers)

#### 1. `checkAndPromptMissingUpdates` (Cron: Twice Daily)
* **Schedule**: `0 8,16 * * *` (8:00 AM & 4:00 PM local time)
* **Timezone**: `Asia/Jerusalem`
* **Logic**:
  1. Scans `/users`.
  2. Filters users who have `enableReminders == true` AND either:
     * `currentLocation == "Unknown"` OR
     * `lastUpdated` is older than 8 hours.
  3. Sends a push notification via Firebase Cloud Messaging (FCM): *"Your family is waiting to know where you are! Update your location."*

#### 2. `flagStaleLocations` (Cron: Hourly)
* **Schedule**: `0 * * * *` (Every hour)
* **Logic**:
  1. Scans `/users`.
  2. Compares the current time with `lastUpdated`.
  3. If `currentTime - lastUpdated > 12 hours`, updates:
     * `currentLocation` ➔ `"Stale"`
     * `targetAngle` ➔ `45` (default physical angle for Stale status)

#### 3. `cleanupExpiredVoiceMessages` (Cron: Daily)
* **Schedule**: `0 2 * * *` (2:00 AM local time)
* **Logic**:
  1. Queries all `/voice_messages` where:
     * `createdAt` is older than 48 hours AND
     * `status == "played"` (preserving unplayed/queued messages).
  2. For each message:
     * Safe-deletes the physical audio file from Cloud Storage (`audioUrl`).
     * Deletes the Firestore document.

#### 4. `clearVisualGreetings` (Cron: Daily)
* **Schedule**: `0 0 * * *` (Midnight local time)
* **Logic**:
  1. Scans `/users`.
  2. For each user with an active `displayGreetingUrl`:
     * Deletes the corresponding image from Cloud Storage.
     * Sets `displayGreetingUrl` to `null` in Firestore, causing LCD screens to revert to regular profile avatars.

## 🔌 HTTP API Endpoints

### 📱 1. Flutter Mobile Client APIs

#### POST `/sendDirectLocationPrompt`
* **Purpose**: Allows one user to request a real-time location update from another.
* **Authentication**: Requires Firebase Auth ID Token (`Authorization: Bearer <ID_TOKEN>`).
* **Request Header**:
  ```http
  Authorization: Bearer eyJhbGciOiJSUzI1NiIs...
  ```
* **Request Body**:
  ```json
  {
    "targetUserId": "user_xyz123"
  }
  ```
* **Logic**:
  1. Verifies caller authentication.
  2. Enforces a **7-minute cooldown** per target user to prevent spamming.
  3. Sends an FCM High-Priority Push Notification to `targetUserId`.
  4. Returns success status.
* **Responses**:
  * `200 OK`: `{ "status": "success", "message": "Prompt sent." }`
  * `429 Too Many Requests`: `{ "status": "error", "message": "Must wait 7 minutes between prompts." }`
  * `401 Unauthorized`: `{ "status": "error", "message": "Invalid token." }`

---

### 🎛️ 2. ESP32 Hardware Client APIs

#### GET `/getClockInitConfig`
* **Purpose**: Fetches the initial clock face mappings and hand assignments on power-on or reset.
* **Authentication**: Requires Hardware secret key header (`X-Clock-ID`).
* **Request Header**:
  ```http
  X-Clock-ID: clock_secret_jwt_or_hash
  ```
* **Response Body (`200 OK`)**:
  ```json
  {
    "locations": [
      { "locationName": "Home", "angle": 180 },
      { "locationName": "Work", "angle": 90 },
      { "locationName": "Technion", "angle": 270 },
      { "locationName": "Stale", "angle": 45 },
      { "locationName": "Unknown", "angle": 0 }
    ],
    "hands": [
      { "handNumber": 1, "userId": "user_dad_123", "userName": "Dad", "currentAngle": 180 },
      { "handNumber": 2, "userId": "user_mom_456", "userName": "Mom", "currentAngle": 90 },
      { "handNumber": 3, "userId": "user_kid_789", "userName": "Leo", "currentAngle": 0 }
    ]
  }
  ```

#### POST `/reportHardwareStatus`
* **Purpose**: ESP32 reports runtime errors, diagnostic status, or hardware failures to the cloud.
* **Authentication**: Requires Hardware secret key header (`X-Clock-ID`).
* **Request Body**:
  ```json
  {
    "speakerStatus": "connected",
    "motorStatus": "jammed",
    "lastErrorCode": "MOTOR_2_STUCK"
  }
  ```
* **Logic**:
  1. Updates the `system_status/main_clock` Firestore document.
  2. If `motorStatus` is `"jammed"` or a critical failure occurs, triggers an instant high-priority push notification to all administrator family members.
* **Response Body (`200 OK`)**:
  ```json
  {
    "status": "success"
  }
  ```
